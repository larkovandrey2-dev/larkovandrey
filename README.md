# Mireya — Сервис мониторинга тревожности

**Mireya** — это умный телеграм-бот для оценки психологического состояния студентов. Проект сочетает в себе классические психологические методики и современные технологии искусственного интеллекта (LLM + ML).

> Проект разработан в рамках курса "Обучение служением" (1 курс).

## Социальная значимость проекта
Согласно данным ВОЗ, в современном студенческом обществе наблюдается устойчивый рост уровня стресса и тревожности, связанный с учебной нагрузкой, неопределённостью будущего и социальным давлением. По статистике ВОЗ, многие молодые люди не имеют навыков управления стрессом и не обращаются за профессиональной поддержкой из-за страха осуждения или недостатка информации. Самым распространённым является именно тревожное расстройство (359 млн человек на 2021 г), также самой частой возрастной категорией, страдающей данным расстройством, являются люди возраста от 18 до 25 лет, то есть по большей части студенты ВУЗов. При этом несмотря на то, что существуют эффективные методы борьбы с тревожностью, помощь получают только ¼ из всех. Повышенный уровень тревожности негативно влияет на работоспособность студента, что провоцирует еще больший стресс и разрушает нервную систему. Таким образом, при невозможности справиться с повышенным уровнем тревожности когнитивные способности человека значительно ухудшаются, а вследствие того, что получение знаний является основной задачей студентов, проблема становится очень серьезной и требует решения. Исходя из важности данной проблемы именно студенты ВУЗа (НИУ МАИ) были выбраны в качестве целевой аудитории проекта.

## Ключевые особенности

### 1. Интеллектуальные опросы
Вместо скучного выбора вариантов ответа, пользователь может отвечать в **свободной форме**.
*   **NLP & LLM (YandexGPT):** Бот понимает естественную речь, выделяет из ответа смысл и конвертирует его в числовые показатели.
*   **Диалоговая валидация:** Если нейросеть не может однозначно интерпретировать ответ, бот задаст уточняющий вопрос, добиваясь понимания, как настоящий собеседник.

### 2. Двухуровневая система оценки
*   **GAD-7 (General Anxiety Disorder-7):** Золотой стандарт скрининга тревожности.
*   **Студенческий профиль:** Специализированный опросник, составленный по рекомендациям психологов для выявления специфических учебных стресс-факторов.

### 3. Машинное обучение (ML)
Для повышения точности результатов используется **Линейная регрессия**:
*   Модель обучена на датасете с **Kaggle**.
*   Позволяет сглаживать неточности интерпретации LLM и учитывать дополнительные факторы при расчете итогового уровня тревожности.

### 4. Личный кабинет и Аналитика
*   **Динамика:** Построение графиков уровня тревожности во времени.
*   **Статистика:** Детальный разбор результатов.
*   **Управление:** Возможность редактировать свои данные.

### 5. Админ-панель
Полноценный интерфейс для управления контентом: создание и редактирование опросов, управление правами доступа.

## Технический стек

Проект построен на микросервисной архитектуре и развернут в облаке.

*   **Язык:** Python 3.10+
*   **Бот:** Aiogram 3.x (асинхронный)
*   **API:** FastAPI
*   **AI/ML:**
    *   **YandexGPT API** — обработка естественного языка.
    *   **Scikit-learn** — модель линейной регрессии.
*   **База данных:** Supabase
*   **Деплой:**
    *   **Docker & Docker Compose** — контейнеризация.
    *   **Yandex Cloud (Compute Cloud)** — хостинг.

## Запуск проекта

Проект полностью докеризирован. Для запуска вам понадобится установленный Docker и Docker Compose.

### 1. Клонирование репозитория
```bash
git https://github.com/larkovandrey2-dev/larkovandrey.git
cd Mireya
```
### 2. Настройка окружения
Создайте файл .env в папке Mireya и заполните его своими данными:


``` Env 
# Telegram Bot Configuration
BOT_TOKEN=your_telegram_bot_token_here

# Supabase Database Configuration
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_SERVICE_KEY=your_supabase_service_key_here

# Yandex Cloud LLM Configuration
YC_API_TOKEN=your_yandex_cloud_api_token_here

# Admin Users (comma-separated Telegram user IDs)
ADMINS=123456789,987654321
```
### 3. Запуск
```bash
docker compose up -d --build
```

## Архитектура решения
* **Frontend (Telegram Bot)**: Отвечает за взаимодействие с пользователем (Polling).  
* **Backend (API)**: Содержит бизнес-логику, обращается к ML-моделям и YandexGPT.  
* **Database**: Хранит профили пользователей, историю ответов и структуру опросов.  
### Алгоритм обработки "Развернутого ответа":

* Пользователь пишет текст ->
* YandexGPT пытается извлечь скор (балл) ->
* Успех: Данные сохраняются для дальнейшего анализа ->
* Неудача: Генерируется уточняющий вопрос -> Пользователь отвечает
